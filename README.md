# Class-2
1. Perfiles de los Atacantes
Cisco clasifica a los delincuentes según su sofisticación y motivación:

Script Kiddies: Aficionados que usan herramientas ya creadas sin entender el código.

Hackers: Divididos en White Hat (Éticos), Black Hat (Maliciosos) y Grey Hat (Intermedios).

Hacktivistas: Atacan por causas políticas o sociales.

Estado-Nación: Ciberataques de alta complejidad financiados por gobiernos para espionaje o sabotaje.

2. Tipos de Malware (Software Malicioso)
Virus: Necesita una acción humana (como abrir un archivo) para ejecutarse y propagarse.

Gusanos (Worms): Se propagan solos por la red explotando vulnerabilidades de red.

Troyanos: Se ocultan dentro de un programa legítimo.

Ransomware: Secuestra datos mediante cifrado y exige un rescate económico.

3. Técnicas de Ataque
Ingeniería Social: Engañar a personas para que revelen claves (Phishing, Pretexting, Baiting).

DoS / DDoS: Ataques de Denegación de Servicio que saturan un servidor para dejarlo inoperativo.

Explotación de Vulnerabilidades: Uso de fallos en el software (como Inyección SQL o Cross-Site Scripting) para robar datos.

Resumen 2: Módulo 4 – El Arte de Proteger los Datos
El enfoque aquí es la defensa. Se presentan los marcos teóricos y las herramientas técnicas necesarias para proteger la información de manera profesional.

1. La Tríada de la CIA (Confidencialidad, Integridad y Disponibilidad)
Es el modelo fundamental de la seguridad informática:

Confidencialidad: Garantizar que solo los autorizados vean los datos (Cifrado).

Integridad: Asegurar que los datos no sean alterados (Hashing).

Disponibilidad: Que el sistema sea accesible cuando se necesite (Redundancia y Backups).

2. Estados de los Datos
La protección debe aplicarse en las tres etapas del ciclo de vida del dato:

Datos en Reposo (In storage): Almacenados en discos o la nube.

Datos en Tránsito (In transmission): Viajando por la red (se protegen con VPN o TLS).

Datos en Proceso (In process): Cuando están siendo usados por la CPU o RAM.

3. Estrategias de Defensa
Defensa en Profundidad: Usar múltiples capas (Firewall, Antivirus, Políticas de Seguridad). Si una falla, la siguiente protege.

Control de Acceso: Uso del modelo AAA (Autenticación, Autorización y Contabilidad).

Principio de Menor Privilegio: Dar a los usuarios solo los permisos necesarios para su tarea específica.

Manejo de Riesgos: Identificar, evaluar y mitigar posibles amenazas antes de que ocurran.
